# gnrtv.cells.26
Pd Toolkit inspired by modular synthesizers within the framework of the algorithmic music, to easily build Generative Algorythmic instruments.

Pd Toolkit for research &amp; educational purposes 
A toolkit programmed in Pd for Sonic Generative Design

gnrtv.cells v.26 edition

1____Install pd version (purr-data / pd vanilla or plugdata) in your OS. 
Repos
purr-data  https://github.com/agraef/purr-data/releases
or pd vanilla https://puredata.info/downloads/pure-data
or plugdata  https://plugdata.org/download.html

2____Once installed check if your Sound Card is working : go to menu media > 'test audio and midi'
and select the 80 value in the 'Output Monitor / Test Gain' area. 
This should emit a sinusoid of 440hz which corresponds to the note (La / A4).

3____gnrtv.cells is a toolkit to easily design sonic generative algoryhtms with pd, 
Structure folders are:
'before.start', In order to understand how 'cells' works click TUTOrial pdf docuiment inside the zip.

'code' where are the main templates. 

'RECS' all live sets can be recorded with the start.rec / stop.rec buttons that appears in the bottom section of 音 Core’s Rack (wich is the main clock and Mixer Block of GNRTV.CELLS). 
Any live performace register can be then recorded which will be automatically stored within the RECS folder.

'sections' the 'guts' of the kit, which are directly called from the main template.

Additionally you can create a new folder in the same level of the rest of folders for example ‘Projects’, in order to store your patches and projects. Also works if you store your projects within the same 'code' folder.
